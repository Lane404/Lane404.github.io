2.1 JSP内置对象展示

名称 描述
appliction(作用域) 类似于ServletContext 生命周期随着tomcat容器的创建而创建 随着tomcat容器创建而销毁
session(作用域) 会话对象 用于设置与获取会话的值
request(作用域) 请求对象 用于设置与获取一次请求的数据
pageContext(作用域) 当前页面上下文对象主要用于获取当前页的数据
response 响应对象
config 配置对象
exeception 异常对象
out 打印对象
page 类似于java中(this)

三 jsp四大内置对象 作用域对象

3.1 罗列

名称 描述
appliction 全局上下文对象(整个web项目都可以获取)
session 会话对象(在一次会话中可以获取)
request 请求对象(在一次请求都是可以获取)
pageContext 当前页面上下文对象(只有在当前页面获取)

3.2 方法

方法名称 方法描述
setAttribute(“键”,”值”); 键值对来存储数据
getAttribute(“键”) 根据键获取值
removeAttribute(“”键””) 根据键来删除键值对

3.3 使用el表达式来获取作用域对象值

1.语法: ${xxxScope.键} xxx表示是四大内置对象的名称
2.作用域范围从小到大
pageContext < request < session < appliction
3.注意点
如果内置对象作用域值下的键名都相同的情况 el表达式 优先获取作用域范围小的值

4.4 动态引入与静态引入区别

动态引入：先翻译两个文件 最后将两个文件内容进行合并 效率低
静态引入: 先合并两个文件 再进行翻译 效率高
